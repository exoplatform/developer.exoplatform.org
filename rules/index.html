eXo Development Rules

This space contains all the rules that each developer must know and respect to be able to contribute on the codebase of eXo projects.

The respect of our quality rules can be monitored by our SonarQube instance

To help projects to continuously increase the level of quality of their developments we organized all rules in levels that will add more and more quality controls.

Each level can :

  • add its own new set of rules over the previous level,
  • increase the severity of a rule already tracked in an inferior level,
